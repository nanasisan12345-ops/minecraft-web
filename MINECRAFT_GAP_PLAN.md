- [done] メッシュ生成の高速化: `buildChunkState` を列単位キャッシュ＋列スタック化して1チャンク約400ms→約37ms（約11倍）に短縮。メッシュワーカーをCPUコア数ぶんのプール（2〜6本）で並列化し、完成メッシュはフレーム予算内で貼ってカクつきを抑制。ロードが約20秒→数秒に。出力はボクセル/メッシュとも最適化前と完全一致。
- [done] 生成/描画の追加軽量化: メインスレッドの `terrainBlockAt` も列単位キャッシュ化（洞窟装飾の重い列走査が主因、プリ生成 約6.4秒→約1.5秒）。草花再構築をフレーム予算ジョブ化して窓移動/設置ごとの約90msスパイクを約10msに。空ブロック種のジオメトリ生成を廃止（共有 `EMPTY_GEO`）。初期プリ生成の予算引き上げ。結果、ロード約1.5秒＋高速移動でも1フレームスパイクが概ね10〜17ms程度に。

## 実装ルール

- 一度に巨大な仕組みを入れず、フェーズ内でも小さく検証する。
- 既存のVite/Three.js分割構成に合わせ、`src/game/parts/*.js` に追加する。
- 構造物・ランドマーク・建物を作るときは、実装前にネット画像検索でMinecraft作例や実物写真を複数確認し、共通するシルエット、比率、屋根/柱/顔/入口などの特徴部位をメモしてから作る。
- 構造物の完了条件は `npm.cmd run check` だけではない。実ブラウザで対象構造物へ移動し、スクリーンショットを参照画像と見比べて、名前を伏せても狙った建物・ランドマークに見えることを確認する。
- 追加後は `npm.cmd run check` を通す。
- `HANDOFF.md` に実装済み内容と次の候補を残す。
