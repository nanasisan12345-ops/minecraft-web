# Minecraft Web

## 開発 / GitHub Pages 公開

このプロジェクトは Vite + Three.js 構成に移行しました。編集は `src/game/parts/*.js` と `src/styles.css` で行い、`src/game.generated.js` は自動生成されるため直接編集しません。

```powershell
npm install
npm run dev
```

ローカル確認: `http://127.0.0.1:5173/`

Pages向けの本番ビルド:

```powershell
npm run build
```

`dist/` がGitHub Pagesへアップロードされる成果物です。`.github/workflows/deploy-pages.yml` を追加済みなので、GitHub側で Pages の Source を「GitHub Actions」にしておけば、`main` へ push したときに自動でビルド/公開されます。`music/` のmp3はビルド時に `dist/music/` へコピーされます。

主な編集場所:

- `src/game/parts/32-world-window.js`: 地形生成範囲、チャンク/窓更新
- `src/game/parts/34-mesh-rebuild.js`: ブロックメッシュ再構築
- `src/game/parts/82-weather-and-loop.js`: 天候とメインループ
- `src/game/parts/60-rave-venues.js`: 音楽会場
- `src/game/parts/70-music.js`: mp3 BGM
- `src/styles.css`: UI表示

| 近くの洞窟入口へ移動 | `F4` |
| 近くの地下遺跡へ移動 | `F6` |
| 近くの村へ移動 | `F7` |
| 近くの廃坑へ移動 | `F8` |
| 近くの地下湖へ移動 | `F9` |
| 近くの峡谷へ移動 | `F10` |
| 富士山へ移動 | `F11` |
| 和風建築へ順送り移動 | `0`（`F3`飛行中） |
| クラフト | `B` |
| かまど | `F` |
| 視点切替 | `N` 一人称 / 三人称 |

最初に画面をクリックするとゲームが開始します。
スポーン地点の周辺は、音楽会場をテストしやすいように木や草花のない平地になっています。
地形確認用に `F3` でデバッグ飛行を切り替えられます。飛行中は `Space` で上昇、`Shift` で下降、`Ctrl` で高速移動できます。`0` は鳥居、水上鳥居、五重塔、茶屋、天守閣、大仏、棚田、東京タワー風タワーを順送りで探して移動します。`F4` は近くの洞窟入口を探して移動します。
通常移動では1ブロック程度の段差を自動で上がれるため、低い起伏は歩いて越えられます。

## クラフト / かまど

`B` でクラフト、`F` でかまどパネルを開きます。
置いた作業台/かまどを右クリックしても、それぞれのパネルを開けます。
ブロック以外に、棒、石炭、粗鉄、粗金、鉄/金インゴット、ダイヤ、道具をインベントリへ持てます。
`Tab` でインベントリ整理画面を開き、ブロック、素材、道具、食料をカテゴリ別に確認できます。

現在の主なレシピ:

- 丸太 -> 板材
- 板材 -> 棒
- 板材 -> 作業台
- 石 -> かまど
- 石炭 + 棒 -> たいまつ
- 板材/石/鉄インゴット + 棒 -> ツルハシ/斧/シャベル
- かまど: 粗鉄 -> 鉄インゴット、粗金 -> 金インゴット、砂 -> ガラス

鉱石を壊すと、石炭、粗鉄、粗金、ダイヤとして入手します。
洞窟内の発光結晶を壊すと、発光結晶の欠片を素材として入手できます。
地下遺跡・廃坑・村などの宝箱は、右クリックでパネルを開き、鉱石・インゴット・食料・たいまつなどの探索報酬を選んで入手できます。中身は宝箱の座標ごとに保存され、空にすると開いた宝箱へ変化します。`F6` で近くの地下遺跡へ移動できます。
村人に照準を合わせて右クリックすると、農家、鍛冶屋、商人、聖職者、見張り、司書、牧場係、旅人ごとの会話と簡易取引パネルを開けます。ベリー、鉱石、石炭、発光結晶の欠片などを、食料、道具、ランタン、建材に交換できます。
村には決定的な名前が付き、入口側に名前看板が生成されます。村の近くにいる間は左上ステータスにも村名が表示されます。
採掘は長押し式で、道具を持っていると対象ブロックを速く掘れます。
道具には耐久値があり、採掘に使うと少しずつ減って、0になると壊れます。
葉や草を壊すとリンゴやベリーが手に入ることがあり、画面右下の食料UIから食べて空腹と体力を回復できます。
画面右下には一人称の手と持ち物が表示され、採掘中はツルハシ/斧/シャベルなどの道具が見えます。
`N` を押すと三人称に切り替わり、ブロック調のプレイヤー本体と持ち物を確認できます。

## 音楽会場モード

`1`〜`8` で、プレイヤーの足元付近に音楽会場を出現させます。
旧キーとして `R/T/Y/U/I/O/P/[` でも同じ会場を切り替えられます。
同じキーをもう一度押すと、その会場を消せます。

| キー | 会場 | 音楽の方向性 | 見た目の方向性 |
|---|---|---|---|
| `1` / `R` | `WAREHOUSE RAVE` | ダークテクノ。サチュレーションの効いた重いキック、オフビートでうねるサブベース、暗いスタブ、深いリバーブ | 暗い倉庫、トラス、スピーカー、青紫系ライト |
| `2` / `T` | `EUROBEAT ROAD` | ユーロビート。速いBPM、16分のオクターブ走りベース、きらびやかなスーパーソウのリード（ディレイ） | 道路ステージ、赤黄ネオン、ゲート状ライト |
| `3` / `Y` | `TRANCE DOME` | アップリフティングトランス。オフビートのゲートベース、巨大なスーパーソウのパッド、アルペジオ（リバーブ＋ディレイ深め） | 半透明ドーム、同心円リング、青紫のクリスタル |
| `4` / `U` | `DANCE FLOOR` | ディスコハウス。ファンキーなシンコペーションベース、7thコードのスタブ、スラップディレイ | 白黒チェック床、ミラーボール、金色の柱 |
| `5` / `I` | `FUTURE BASS SKY` | フューチャーベース。明るいコード、声ネタ風シンセ、サブドロップ、ビルドアップのライザー | 浮遊島、菱形フロア、ピンクとシアンのリング |
| `6` / `O` | `DRUM BASS TUNNEL` | ドラムンベース。高速ブレイクビーツ、ワブルベース、強いサブ、暗いフィル | 緑発光のトンネル、スピーカー壁、細いレーザー |
| `7` / `P` | `LOFI HOUSE LOUNGE` | ローファイハウス。遅めのBPM、柔らかいコード、ゆるいキック、空気感のあるノイズ | 月明かり、ソファ、丸いラウンジ床、控えめな照明 |
| `8` / `[` | `DUBSTEP LAB` | ダブステップ。ハーフタイムのドラム、うねるグロウルベース、金属的なスクリーチ | 暗い実験室、強いマゼンタ照明、細かいレーザー |

会場BGMは `music/` に置いた mp3 を再生します。mp3 が無い、または再生に失敗した会場は無音のままビジュアルと拍同期だけ動きます。
著作権付き楽曲は入れていません。

画面左下には会場キーの小さな一覧が表示され、現在出ている会場は光って分かるようになっています。
この一覧はクリック/タップでも会場を切り替えられます。
左上のステータスには、会場名、曲番号、BPM、NPCの入場状況も表示されます。

サビやドロップでは、観客NPCが波のようにジャンプして手を上げ、外側へ広がります。
歓声のような光粒子も円形に飛びます。
会場内で `Z/X/C/V` を押すとプレイヤーダンスになり、近くのNPCがこちらを向いて真似します。
DJ操作やダンス、サビの盛り上がりで右上の `CROWD HYPE` ゲージが上がります。
満タンになると `HYPE MAX` になり、観客リアクション、照明、クラッシュ音、上空の花火がさらに派手になります。
`K` のクラッシュ操作でも小さめの花火が打ち上がります。
DJ操作やダンスを続けると `GROOVE` コンボが伸び、ゲージ上昇量が少し増えます。
操作時には画面フラッシュ、GOOD系のポップ表示、床を走る光のショックウェーブも出ます。
各会場には拍同期ギミックがあり、Future Bassでは浮遊島、DNBではトンネルの光、Lofiでは月やランプがふわっと動きます。
床タイル、ゲート、音ブロック風の小さなブロックもキックやサビに合わせて伸びたり光ったりします。
会場を出した後に `Q/E/J/K/L` を押すと、曲送り、フィルター、クラッシュ、ループロールをDJのように操作できます。

各会場は、コンプレッサー／コンボリューションリバーブ／テンポ同期ディレイの FX バスと、
キックに合わせたサイドチェイン（ポンプ感）を共通で持ち、ジャンルごとに BPM・リズム・
ベース音色・コード進行・リード／パッドを丸ごと変えて差別化しています。

## 技術メモ

- メイン実装は `src/game/parts/*.js` です。`src/game.generated.js` は `npm run assemble` で生成されます。
- 3D 描画は Vite 経由で Three.js を読み込んでいます。
- 地形はプレイヤー周辺を動的生成する方式です。
- ブロック表示は種類ごとに `InstancedMesh` を使っています。
- ブロック破壊/設置はレイキャストで判定しています。
- RAVE/音楽会場関連の主な実装は `src/game/parts/60-rave-venues.js` と `src/game/parts/70-music.js` です。
- 会場BGMの mp3 検出/再生は `70-music.js`、会場の形・拍同期・DJ操作は `60-rave-venues.js` が中心です。

## 引き継ぎメモ

- 現在のブランチは `main`、公開先は GitHub Pages です。
- 変更後は `npm.cmd run check` を通してから push してください。
- 文字化けしやすいので、README や表示文言を編集するときは UTF-8 として扱ってください。
- 未追跡フォルダ `gikopoi2/` と `hallucinate/` はこのプロジェクトの変更としては触らないでください。
- 会場を増やす場合は、`RAVE_KEY_BINDINGS`、`RAVE_VENUES`、`RAVE_FX`、`step***()` シーケンサー、会場生成ロジックをセットで更新してください。
- 音楽ジャンルを増やす場合は、既存曲の微変更ではなく、BPM、リズム、ベース音色、リード/パッド、コード進行、会場の形をまとめて変えると差が出ます。
