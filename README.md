# Minecraft Web

## 開発 / GitHub Pages 公開

このプロジェクトは Vite + Three.js 構成に移行しました。編集は `src/game/parts/*.js` と `src/styles.css` で行い、`src/game.generated.js` は自動生成されるため直接編集しません。

```powershell
npm install
npm run dev
```

ローカル確認: `http://127.0.0.1:5173/`

Pages向けの本番ビルド:

```powershell
npm run build
```

`dist/` がGitHub Pagesへアップロードされる成果物です。`.github/workflows/deploy-pages.yml` を追加済みなので、GitHub側で Pages の Source を「GitHub Actions」にしておけば、`main` へ push したときに自動でビルド/公開されます。`music/` のmp3はビルド時に `dist/music/` へコピーされます。

主な編集場所:

- `src/game/parts/32-world-window.js`: 地形生成範囲、チャンク/窓更新
- `src/game/parts/34-mesh-rebuild.js`: ブロックメッシュ再構築
- `src/game/parts/82-weather-and-loop.js`: 天候とメインループ
- `src/game/parts/60-rave-venues.js`: 音楽会場
- `src/game/parts/70-music.js`: mp3 BGM
- `src/styles.css`: UI表示

## マップ生成

地形生成は `src/game/parts/32-world-window.js` にまとまっています。現在は座標シードから以下を自動生成します。

- バイオーム: 草原 / 森林 / 砂漠 / 高地 / 雪原 / 沼地 / ジャングル / 火山
- バイオーム別の高さ、表層ブロック、木の密度、草花の密度
- 高地/雪原は探索しやすいよう、高さ差を抑えた山地として生成
- ジャングルは木が密で高く、沼地は水辺が多く、火山は高所に溶岩クレーター
- 自然ディテール: 岩、倒木、砂漠の枯れ木とサボテン、沼の水草
- 夜は草原/森林/沼地/ジャングルでプレイヤー周囲に蛍が舞う
- 小さな池と浅く蛇行する水路
- 連続して流れる川（うねる水路）と、深い峡谷（壁に鉱石、底に川や溶岩）。峡谷の壁には滝が落ちる
- 地下空洞と洞窟入口
- 洞窟内の発光結晶、鍾乳石、入口付近の小さな発光目印
- 地下遺跡ダンジョン（石レンガの部屋、ランタン、宝箱、地上への階段入口）
- 廃坑（木の支柱トンネル、たいまつ、露出鉱石、溶岩、終端の宝箱、地上入口）
- 地下湖（水または溶岩の大空洞、発光結晶、地上への階段）
- 深層ほど出やすいダイヤ/金、地下の溶岩（触れるとダメージ）、サボテン（近づくとダメージ）
- 村クラスタ（井戸、家、畑、道、街灯、鍛冶屋、市場。家や鍛冶屋・市場に宝箱）。村人が住みつく
- 小屋、廃墟、石塔、砂漠の小神殿
- 構造物周辺や草原に出る村人風の旅人NPC

左上ステータスに現在位置のバイオーム名と地形チャンク数を表示します。

Three.js で作ったブラウザ版の 3D ブロックゲームです。
`index.html` だけで動く構成で、GitHub Pages で公開しています。

最初に画面をクリックするとゲームが開始します。
スポーン地点の周辺は、音楽会場をテストしやすいように木や草花のない平地になっています。
地形確認用に `F3` でデバッグ飛行を切り替えられます。飛行中は `Space` で上昇、`Shift` で下降、`Ctrl` で高速移動できます。`F4` は近くの洞窟入口を探して移動します。
通常移動では1ブロック程度の段差を自動で上がれるため、低い起伏は歩いて越えられます。

## クラフト / かまど

`B` でクラフト、`F` でかまどパネルを開きます。
置いた作業台/かまどを右クリックしても、それぞれのパネルを開けます。
ブロック以外に、棒、石炭、粗鉄、粗金、鉄/金インゴット、ダイヤ、道具をインベントリへ持てます。
`Tab` でインベントリ整理画面を開き、ブロック、素材、道具、食料をカテゴリ別に確認できます。

現在の主なレシピ:

- 丸太 -> 板材
- 板材 -> 棒
- 板材 -> 作業台
- 石 -> かまど
- 石炭 + 棒 -> たいまつ
- 板材/石/鉄インゴット + 棒 -> ツルハシ/斧/シャベル
- かまど: 粗鉄 -> 鉄インゴット、粗金 -> 金インゴット、砂 -> ガラス

鉱石を壊すと、石炭、粗鉄、粗金、ダイヤとして入手します。
洞窟内の発光結晶を壊すと、発光結晶の欠片を素材として入手できます。
地下遺跡・廃坑・村などの宝箱は、右クリックで開けると鉱石・インゴット・食料・たいまつなどの探索報酬がまとめて手に入ります（壊しても入手できます）。`F6` で近くの地下遺跡へ移動できます。
採掘は長押し式で、道具を持っていると対象ブロックを速く掘れます。
道具には耐久値があり、採掘に使うと少しずつ減って、0になると壊れます。
葉や草を壊すとリンゴやベリーが手に入ることがあり、画面右下の食料UIから食べて空腹と体力を回復できます。
画面右下には一人称の手と持ち物が表示され、採掘中はツルハシ/斧/シャベルなどの道具が見えます。
`N` を押すと三人称に切り替わり、ブロック調のプレイヤー本体と持ち物を確認できます。

## 音楽会場モード

`R/T/Y/U/I/O/P` で、プレイヤーの足元付近に音楽会場を出現させます。
同じキーをもう一度押すと、その会場を消せます。

| キー | 会場 | 音楽の方向性 | 見た目の方向性 |
|---|---|---|---|
| `R` | `WAREHOUSE RAVE` | ダークテクノ。サチュレーションの効いた重いキック、オフビートでうねるサブベース、暗いスタブ、深いリバーブ | 暗い倉庫、トラス、スピーカー、青紫系ライト |
| `T` | `EUROBEAT ROAD` | ユーロビート。速いBPM、16分のオクターブ走りベース、きらびやかなスーパーソウのリード（ディレイ） | 道路ステージ、赤黄ネオン、ゲート状ライト |
| `Y` | `TRANCE DOME` | アップリフティングトランス。オフビートのゲートベース、巨大なスーパーソウのパッド、アルペジオ（リバーブ＋ディレイ深め） | 半透明ドーム、同心円リング、青紫のクリスタル |
| `U` | `DANCE FLOOR` | ディスコハウス。ファンキーなシンコペーションベース、7thコードのスタブ、スラップディレイ | 白黒チェック床、ミラーボール、金色の柱 |
| `I` | `FUTURE BASS SKY` | フューチャーベース。明るいコード、声ネタ風シンセ、サブドロップ、ビルドアップのライザー | 浮遊島、菱形フロア、ピンクとシアンのリング |
| `O` | `DRUM BASS TUNNEL` | ドラムンベース。高速ブレイクビーツ、ワブルベース、強いサブ、暗いフィル | 緑発光のトンネル、スピーカー壁、細いレーザー |
| `P` | `LOFI HOUSE LOUNGE` | ローファイハウス。遅めのBPM、柔らかいコード、ゆるいキック、空気感のあるノイズ | 月明かり、ソファ、丸いラウンジ床、控えめな照明 |

音は外部音源ファイルではなく、Web Audio API のシンセでリアルタイム生成しています。
著作権付き楽曲は入れていません。

画面左下には会場キーの小さな一覧が表示され、現在出ている会場は光って分かるようになっています。
この一覧はクリック/タップでも会場を切り替えられます。
左上のステータスには、会場名、曲番号、BPM、NPCの入場状況も表示されます。

サビやドロップでは、観客NPCが波のようにジャンプして手を上げ、外側へ広がります。
歓声のような光粒子も円形に飛びます。
会場内で `Z/X/C/V` を押すとプレイヤーダンスになり、近くのNPCがこちらを向いて真似します。
DJ操作やダンス、サビの盛り上がりで右上の `CROWD HYPE` ゲージが上がります。
満タンになると `HYPE MAX` になり、観客リアクション、照明、クラッシュ音、上空の花火がさらに派手になります。
`K` のクラッシュ操作でも小さめの花火が打ち上がります。
DJ操作やダンスを続けると `GROOVE` コンボが伸び、ゲージ上昇量が少し増えます。
操作時には画面フラッシュ、GOOD系のポップ表示、床を走る光のショックウェーブも出ます。
各会場には拍同期ギミックがあり、Future Bassでは浮遊島、DNBではトンネルの光、Lofiでは月やランプがふわっと動きます。
床タイル、ゲート、音ブロック風の小さなブロックもキックやサビに合わせて伸びたり光ったりします。
会場を出した後に `Q/E/J/K/L` を押すと、曲送り、フィルター、クラッシュ、ループロールをDJのように操作できます。

各会場は、コンプレッサー／コンボリューションリバーブ／テンポ同期ディレイの FX バスと、
キックに合わせたサイドチェイン（ポンプ感）を共通で持ち、ジャンルごとに BPM・リズム・
ベース音色・コード進行・リード／パッドを丸ごと変えて差別化しています。

## 技術メモ

- メイン実装は `index.html` のみです。
- 3D 描画は Three.js を CDN から読み込んでいます。
- 地形はプレイヤー周辺を動的生成する方式です。
- ブロック表示は種類ごとに `InstancedMesh` を使っています。
- ブロック破壊/設置はレイキャストで判定しています。
- RAVE/音楽会場関連の主な実装は `RAVE`、`RAVE_KEY_BINDINGS`、`RAVE_VENUES`、`RAVE_FX`、`buildRaveVenue()`、`raveToggle()`（FXバス構築）、`raveUpdate()` 周辺です。
- 音楽は `scheduleStep()` がジャンル別の `stepTechno()` / `stepEurobeat()` / `stepTrance()` / `stepDisco()` / `stepFutureBass()` / `stepDrumBass()` / `stepLofiHouse()` に振り分けます。コード進行は `PROG`、リードは `EUMEL` などにあります。
- 音源（シンセ）は `vKick` / `vBass` / `vSuper`（スーパーソウ）/ `vStab` / `vChord` / `vClap` / `vSnare` / `vHat` など。FX 関連は `makeReverbIR` / `makeShaper` / `sendTo` / `duckTrigger`（サイドチェイン）です。

## 引き継ぎメモ

- 現在のブランチは `main`、公開先は GitHub Pages です。
- 変更後は `index.html` を構文チェックしてから push してください。
- 文字化けしやすいので、README や表示文言を編集するときは UTF-8 として扱ってください。
- 未追跡フォルダ `gikopoi2/` と `hallucinate/` はこのプロジェクトの変更としては触らないでください。
- 会場を増やす場合は、`RAVE_KEY_BINDINGS`、`RAVE_VENUES`、`RAVE_FX`、`step***()` シーケンサー、会場生成ロジックをセットで更新してください。
- 音楽ジャンルを増やす場合は、既存曲の微変更ではなく、BPM、リズム、ベース音色、リード/パッド、コード進行、会場の形をまとめて変えると差が出ます。

## 確認コマンド

PowerShell で以下を実行すると、`index.html` 内の module script の構文だけを確認できます。

```powershell
@'
const fs = require('fs');
const html = fs.readFileSync('index.html', 'utf8');
const m = html.match(/<script type="module">([\s\S]*?)<\/script>/);
if (!m) throw new Error('module script not found');
let code = m[1].replace(/import \* as THREE from 'three';/, 'const THREE = {};');
new Function(code);
console.log('module syntax ok');
'@ | node -
```

ローカルで確認する場合:

```powershell
python -m http.server 8123 --bind 127.0.0.1
```

その後、ブラウザで `http://127.0.0.1:8123/` を開きます。
